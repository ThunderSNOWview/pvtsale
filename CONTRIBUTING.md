# Contributing

Thanks for helping improve pvtsale.

## Getting started

1. Clone the repository and install dependencies in the area you are changing (`ui/` and/or `contracts/`).
2. Run the app or compile contracts locally before opening a pull request.
3. Keep changes focused: one logical change per PR is easier to review.

## Pull requests

- Describe **what** changed and **why** (user-visible behavior, deploy impact, or contract risk).
- Link related issues or discussions when applicable.
- Do not commit `.env` files, private keys, or other secrets (see `.gitignore`).

## Code style

Match existing patterns in the touched package (TypeScript/React in `ui/`, Solidity/Hardhat in `contracts/`). Prefer small, readable diffs over wide refactors unless agreed beforehand.

## Conduct

All participants are expected to follow the [Code of Conduct](CODE_OF_CONDUCT.md).
